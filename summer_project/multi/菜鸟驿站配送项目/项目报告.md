# 菜鸟驿站配送系统 - 项目报告

## 一、 数据结构设计与复杂度分析

本项目在核心数据结构上采用了分离式面向对象设计，重点保障图计算的高效性。

### 1. 核心数据结构
* **Graph (图结构)**：采用了**邻接表 (Adjacency List)** 作为底层存储，用 std::vector<std::vector<Edge>> 实现。相比于邻接矩阵，它更适合现实世界中比较稀疏的道路网络。
  * **空间复杂度**：O(V + E)，其中 V 为节点数，E 为边数。
* **MinHeap (小根堆)**：为了优化最短路径算法，我们手动实现了一个带有动态索引映射 (Index Mapping) 的小根堆。它不仅支持快速插入，还能在 O(log V) 时间内更新指定节点的值。
* **CircularQueue (循环队列)**：我们手动封装了一个支持泛型的循环队列。通过动态数组和 head/tail 双指针取模运算实现。它用于在驿站维护待派发的包裹队列，入队 (enqueue) 和出队 (dequeue) 操作都严格保证 O(1) 的时间复杂度。
* **distCache (距离缓存表)**：为了避免在包裹配送模拟过程中重复计算任意两点间的最短路径，我们引入了缓存机制。首次查询某点 u 到其它点距离时，会执行一次 Dijkstra 算法，并将结果缓存在一个大小为 V 的一维数组中。后续查询直接 O(1) 返回。

### 2. 核心算法时间复杂度
* **Dijkstra 寻路 (单源最短路径)**：配合 MinHeap，时间复杂度为 O((V + E) log V)。
* **全局缓存构建**：在最坏情况下（访问所有节点对），需要以每个节点为起点跑一次 Dijkstra，总时间复杂度为 O(V * (V + E) log V)。

## 二、 各配送任务的思路与正确性说明

### T1 - 最短路径算法 (Shortest Path)
* **思路**：基于经典的 Dijkstra 算法实现，利用优先队列（小根堆）进行松弛操作，求出菜鸟驿站（节点 0）到指定目的地的最短物理距离。同时记录前驱节点数组 prev 方便路径的回溯。
* **正确性说明**：由于道路权重（距离）恒定大于 0，Dijkstra 算法能从数学上保证绝对最优的单源最短路径。

### T2 - 最小化不满度 (Minimize Dissatisfaction)
* **难点**：包裹的不满意度定义为 送达时间 - 到站时间。由于车辆必须等待当前批次中最晚到达的包裹，盲目等待或绕远路都会导致车上所有包裹的不满度产生较大累积。
* **算法思路 (基于时间窗口的贪心批处理)**：
  1. **动态耐心度分批**：首先将所有包裹按到站时间升序排序。小车在容量允许范围内尝试装载包裹，但引入了**“延误代价惩罚”**机制。若为了等待下一个包裹，会导致车上已装载的 k 个包裹产生的总延误时间（Penalty = k * Δt）超过**额外跑一趟往返的平均物理时间代价**（Threshold = `(2 * 驿站到所有节点的平均最短距离) / 车辆速度`），小车将直接中断装货，果断发车！这避免了为了等一个包裹而让一车包裹陪着干等的雪崩效应。
  2. **最近邻路由 (Nearest Neighbor)**：发车后，为了最小化车上所有包裹的送达时间, 我们在每个卸货点都贪心地选择**物理距离最近**的下一个目的地进行派送。
* **有效性说明**：“动态耐心度分批”机制通过引入真实的物理参数（距离、速度），在“车辆装载率”与“整体延误时间”之间建立了极具科学依据的平衡；而在微观派送阶段，“最近邻优先”能有效地压低整体的送达时间，两相结合使得最终的不满意度之和大幅下降，更加接近真正的最优解。

### T3 - 综合配送成本优化 (Transport Cost Optimization)
* **难点**：本任务引入了真实的物理耗能模型——行驶成本不仅和距离有关，还与**小车自重 + 当前车上剩余包裹总重**成正比。这意味着如果车上一直装着一个极重的包裹到处跑，哪怕走的都是最短路，总成本也会因为巨大的重量乘积而爆炸。同时还要兼顾包裹的截止时间。
* **算法思路 (动态载重卸载启发式)**：
  1. ** ddl优先装车**：使用最早截止时间优先 对驿站内的包裹进行排序，将最紧急的包裹优先推入 `CircularQueue`，保证尽量不发生超时。
  2. **多维复合评分出车**：在决定车辆下一个卸货点时，我们抛弃了单纯的最近邻策略，独创了复合动态评分公式：`Score += Weight / (Distance + ε)`。
* **有效性说明**：这个公式是该算法的灵魂。它会促使车辆在派送时，优先把单位距离内消耗重量最多的包裹送达。这种“尽早卸载重物”的贪心策略，在在解决这道任务时，相比传统的最短路算法能带来一定的成本缩减。

### T4 - 车辆返程与 TSP 优化 (Return Logistics)
* **思路**：针对少量包裹，可以暴力求解；当节点数量增加时，我们先用贪心求一个局部次优解，然后采用 **2-Opt 邻域搜索启发式算法**。通过不断尝试翻转当前路径中的两条边，如果总距离减少则接受该翻转。为了避免陷入恶意数据的死循环，我们设置了 2000 次的最大迭代限制。
* **正确性说明**：2-Opt 是一种局部优化算法，它能够极大地优化初态的随机路径，保证最终路线不出现“交叉折返”，获得一个高质量的局部最优解。

### T5 - 双车协同配送 (Two-Car Coordination)
* **思路**：采用 **Cluster-First, Route-Second (先聚类，后路径规划)** 的思想。首先计算出所有包裹中最远的两个点作为“锚点”，然后利用类似 K-Means 的思想将所有包裹分配给距离锚点最近的车。分配完毕后，两辆车分别独立运行 T3 的调度算法。
* **正确性说明**：通过空间划界隔离，避免了两辆车前往同一个区域发生路线重叠导致的资源浪费。

## 三、 T6 附加题对比

为了探究贪心算法的局限性，我们实现了三种不同的贪心策略，并在 3 组测试集上进行了对比验证（每组数据均包含 6 个包裹，包含不同类型的诱导陷阱节点）。

### 策略定义
1. **最近优先 (Nearest Neighbor)**：纯看距离，Score = 1.0 / dist。
2. **最重优先 (Heaviest First)**：纯看重量，Score = weight。
3. **综合性价比 (Composite)**：我们的 T3 策略，Score = weight / dist。

### 测试结果与缺陷分析

**Dataset 1 (Anti-NN) - 最近优先陷阱测试**
[Nearest Neighbor] Cost: 1545, Timeouts: 0
[Heaviest First]   Cost: 1528, Timeouts: 0
[Composite]        Cost: 1528, Timeouts: 0

* **缺陷分析**：“最近优先”策略被设计好的一堆近距离、零重量（0kg）包裹所诱导，进入了死胡同。随后不得不满载着 9kg 的重型包裹原路折返，导致了巨额的载重移动惩罚成本。

**Dataset 2 (Anti-Heavy) - 最重优先陷阱测试**
[Nearest Neighbor] Cost: 1165, Timeouts: 0
[Heaviest First]   Cost: 1285, Timeouts: 0
[Composite]        Cost: 1165, Timeouts: 0

* **缺陷分析**：“最重优先”被放在极远处的一个重包裹所吸引，完全忽略了眼前原本可以顺路批量送达的集群包裹，导致空跑了大量距离。

**Dataset 3 (Anti-Composite) - 综合比例陷阱测试**
[Nearest Neighbor] Cost: 1145, Timeouts: 0
[Heaviest First]   Cost: 1755, Timeouts: 0
[Composite]        Cost: 1755, Timeouts: 0

* **缺陷分析**：综合性价比虽然看起来最聪明，但当远距离包裹的重量基数过大时（产生较高的比值假象），综合策略会被算法欺骗，做出了和“最重优先”一样的错误远征决策。反而是最“短视”的最近优先策略因为瞎猫碰见死耗子，规避了这个长途奔袭陷阱。

### T6 实验总结
通过以上的数学验证和模拟数据跑测，得出了一个非常关键的结论：**在带有容量限制和权重的车辆路径规划问题，没有任何一种单一维度的贪心算法能够做到全局最优**。每种启发式策略都有其难以避免的局部盲区，可能需要更加宏观的算法。
