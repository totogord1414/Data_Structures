# 菜鸟驿站配送调度系统 · 项目材料包

这是课程项目的**学生材料包**。里面**只有需求文档和数据，没有任何代码**——
数据结构、程序结构、算法，全部由你自己设计和实现。

## 包内容

```
菜鸟驿站配送项目/
├── 项目说明.md          ← 先读这个：项目要求、要实现的任务、评分标准
├── 数据格式说明.md       ← 再读这个：输入文件 map/packages/car 的格式（你的程序要按此读入）
├── 测试数据/
│   ├── 示例/            6 节点小地图 + 预期结果.md（自检你的最短路和成本计算）
│   ├── 测试1/           12 节点
│   ├── 测试2/           16 节点
│   └── 测试3/           20 节点
└── README.md           （本文件）
```

每组数据都是三个纯文本文件：`map.txt`（地图）、`packages.txt`（包裹）、`car.txt`（小车）。

## 怎么开始

1. 读 **`项目说明.md`**，搞清楚要做什么、怎么评分；
2. 读 **`数据格式说明.md`**，先写好“读入数据”的代码；
3. 用 **`测试数据/示例/`** 跑通，对照 `预期结果.md` 核对你的最短路与成本计算是否正确；
4. 依次完成 T1（最短路）、T2（不满意度）、T3（成本 + 超时）；
5. 用 `测试1/2/3` 跑出结果、整理进报告；有余力再做选做与可视化。

## 记住

- **我们不提供代码框架**：请自己设计数据结构与程序结构，这本身就是考核内容；
- 你的程序必须能读入《数据格式说明》规定的格式——评分会用**同样格式、不同内容**的数据来跑；
- 鼓励用 AI，但要能讲清自己写的每一行，并如实记录在 `AI_USAGE.md` 里。

需要环境：一个 C++17 编译器（如 `g++` / `clang++`）。祝2025级的每一位同学的包裹都会顺利运输！🚚
