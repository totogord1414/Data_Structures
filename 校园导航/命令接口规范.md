# 命令接口规范

> **本文档是评测接口的强制规范，必须遵守。**
>
> 项目的代码骨架仅作参考（详见《项目说明》§8），你的程序内部如何组织、用什么数据结构、要不要在 CLI 之外再做交互界面/Web/GUI，**完全自由**。但**为方便助教批量评测**，所有同学的程序都必须提供一个 **CLI 批处理入口**，按本规范定义的命令名、参数顺序、输出格式实现。
>
> 也就是说：
>
> - **内部实现自由**——风格、数据结构、模块划分由你定；
> - **评测接口强制**——命令名、关键字（`OK`、`ERROR`、`PATH`、`MST`、`CRITICAL` 等）、字段顺序、排序规则必须与本规范完全一致。

## 概述

程序须能从标准输入按行读取命令，每执行一条命令按规范的格式输出到标准输出（每条命令一行或多行）。遇到 `QUIT` 命令时退出。空行直接跳过。未知命令输出 `ERROR unknown_command`。

评测时使用：

```bash
./CampusNavigation < command.txt > answer.txt
```

---

## 通用输出规则

- 成功的维护操作输出 `OK`
- 失败时输出 `ERROR <reason>`，常见 reason 包括：
  - `place_not_found` — 地点不存在
  - `place_already_exists` — 地点已存在
  - `road_not_found` — 道路不存在
  - `road_already_exists` — 道路已存在
  - `invalid_field` — 不支持的字段名
  - `unknown_command` — 未知命令

## 排序规则

为保证助教批量评测时输出与参考答案易于对照，必须遵守以下排序要求：

1. `QUERY_CATEGORY` 输出的 place_id 按**字典序升序**。
2. `ADJ` 输出的邻接点按邻接点 place_id **字典序升序**。
3. `COMPONENTS` 中的规模列表按**降序**输出。
4. `MST` 中的边按 `(min(u,v), max(u,v))` **字典序**排序输出。
5. `CRITICAL` 中关键节点按字典序，关键边按 `(min(u,v), max(u,v))` 字典序排序输出。

## 命令详细说明

### 1. LOAD

加载地点和道路数据文件，替换当前图。

```
LOAD <places_file> <roads_file>
```

**参数：**
- `places_file`：地点 CSV 文件路径
- `roads_file`：道路 CSV 文件路径

**输出：** `OK` 或 `ERROR <reason>`

**示例：**
```
输入：LOAD places.csv roads.csv
输出：OK
```

---

### 2. SAVE

将当前图保存到 CSV 文件。

```
SAVE <places_out_file> <roads_out_file>
```

**输出：** `OK`

**示例：**
```
输入：SAVE output_places.csv output_roads.csv
输出：OK
```

---

### 3. QUERY_PLACE

查询地点完整信息。

```
QUERY_PLACE <place_id>
```

**输出：**
```
PLACE <place_id> <display_name> <category> <stay_time> <open_time> <close_time>
```

若不存在：`ERROR place_not_found`

**示例：**
```
输入：QUERY_PLACE P0001
输出：PLACE P0001 Library Teaching 30 08:00 22:00
```

---

### 4. QUERY_CATEGORY

按类别查询所有地点。

```
QUERY_CATEGORY <category>
```

**输出：**
```
CATEGORY <category> <count> <id1> <id2> ...
```

place_id 按字典序升序。若无匹配：`CATEGORY <category> 0`

**示例：**
```
输入：QUERY_CATEGORY Teaching
输出：CATEGORY Teaching 2 P0001 P0005
```

---

### 5. ADJ

查询某地点的相邻道路。

```
ADJ <place_id>
```

**输出：**
```
ADJ <place_id> <count> <neighbor1>:<distance>:<walk_time>:<status> <neighbor2>:...
```

邻接点按 place_id 字典序升序。若地点不存在：`ERROR place_not_found`

**示例：**
```
输入：ADJ P0001
输出：ADJ P0001 3 P0002:200:3:open P0003:150:2:open P0005:400:5:open
```

---

### 6. ADD_PLACE

新增地点。

```
ADD_PLACE <place_id> <display_name> <category> <stay_time> <open_time> <close_time>
```

**输出：** `OK` 或 `ERROR place_already_exists`

**示例：**
```
输入：ADD_PLACE P0007 Cafe Dining 20 07:00 22:00
输出：OK
```

---

### 7. DELETE_PLACE

删除地点（同时删除所有与该地点相连的道路）。

```
DELETE_PLACE <place_id>
```

**输出：** `OK` 或 `ERROR place_not_found`

---

### 8. UPDATE_PLACE

修改地点的某个字段。

```
UPDATE_PLACE <place_id> <field> <value>
```

**支持的字段：** `display_name`、`category`、`stay_time`、`open_time`、`close_time`

**输出：** `OK`、`ERROR place_not_found` 或 `ERROR invalid_field`

**示例：**
```
输入：UPDATE_PLACE P0001 stay_time 45
输出：OK
```

---

### 9. ADD_ROAD

新增道路。

```
ADD_ROAD <from_id> <to_id> <distance> <walk_time> <status>
```

**输出：** `OK`、`ERROR place_not_found` 或 `ERROR road_already_exists`

**示例：**
```
输入：ADD_ROAD P0001 P0007 120 2 open
输出：OK
```

---

### 10. DELETE_ROAD

删除道路。

```
DELETE_ROAD <from_id> <to_id>
```

**输出：** `OK` 或 `ERROR road_not_found`

---

### 11. UPDATE_ROAD

修改道路的某个字段。

```
UPDATE_ROAD <from_id> <to_id> <field> <value>
```

**支持的字段：** `distance`、`walk_time`、`status`

**输出：** `OK`、`ERROR road_not_found` 或 `ERROR invalid_field`

---

### 12. CLOSE_ROAD

关闭道路（将 status 设为 closed）。

```
CLOSE_ROAD <from_id> <to_id>
```

**输出：** `OK` 或 `ERROR road_not_found`

---

### 13. OPEN_ROAD

打开道路（将 status 设为 open）。

```
OPEN_ROAD <from_id> <to_id>
```

**输出：** `OK` 或 `ERROR road_not_found`

---

### 14. COMPONENTS

计算连通分量（仅考虑 open 边）。

```
COMPONENTS
```

**输出：**
```
COMPONENTS <count> SIZES <s1> <s2> ...
```

规模按降序输出。

**示例：**
```
输入：COMPONENTS
输出：COMPONENTS 2 SIZES 5 3
```

---

### 15. SHORTEST

计算两点间最短路径。

```
SHORTEST <from_id> <to_id> <DIST|TIME>
```

**参数：**
- `DIST`：按 distance 最短
- `TIME`：按 walk_time 最短

**输出：**
```
PATH <DIST|TIME> <total_cost> NODES <id1> <id2> ...
```

若不可达：`NO_PATH`

若地点不存在：`ERROR place_not_found`

**示例：**
```
输入：SHORTEST P0001 P0006 DIST
输出：PATH DIST 680 NODES P0001 P0003 P0005 P0006
```

---

### 15'. TIMED_SHORTEST

在给定时刻（HH:MM）下计算两点间最短路径。

```
TIMED_SHORTEST <from_id> <to_id> <time> <DIST|TIME>
```

**参数：**
- `time`：查询时刻，HH:MM 格式
- `DIST`：按 `distance` 最短
- `TIME`：按 `walk_time` 最短

**约束：**
- 仅走 `status = open` 的道路；
- 路径上所有地点（含起点和终点）必须在 `time` 时开放
  （即 `open_time ≤ time ≤ close_time`）；
- 若起点或终点此时未开放，直接视为不可达。

**输出：** 与 `SHORTEST` 相同格式

若不可达：`NO_PATH`

若地点不存在：`ERROR place_not_found`

**示例：**
```
输入：TIMED_SHORTEST P0001 P0005 19:00 DIST
输出：NO_PATH
（P0005 在 19:00 已关闭 —— close_time=18:00）
```

---

### 16. MUST_PASS

必经点路径规划（必须按给定顺序经过指定地点）。

```
MUST_PASS <from_id> <to_id> <DIST|TIME> <k> <p1> <p2> ... <pk>
```

**参数：**
- `k`：必经点个数
- `p1`~`pk`：必须按此顺序依次经过的地点

**输出：** 与 SHORTEST 相同格式

**示例：**
```
输入：MUST_PASS P0001 P0007 DIST 2 P0003 P0006
输出：PATH DIST 760 NODES P0001 P0002 P0003 P0005 P0006 P0007
```

---

### 17. MST

计算最小生成树（按 distance，仅考虑 open 边）。

```
MST
```

**输出：**
```
MST <total_distance> EDGES <u1>-<v1>:<w1> <u2>-<v2>:<w2> ...
```

边按 `(min(u,v), max(u,v))` 字典序排序。每条边中 u 和 v 取 place_id 字典序较小者在前。

若图不连通：`DISCONNECTED`

**示例：**
```
输入：MST
输出：MST 980 EDGES P0001-P0002:200 P0001-P0003:150 P0003-P0005:180 P0004-P0005:100 P0005-P0006:350
```

---

### 18. CRITICAL

关键节点与关键边分析（仅考虑 open 边）。

- **关键节点**：删去后图的连通分量数增加的顶点；
- **关键边**：删去后图的连通分量数增加的边。

```
CRITICAL
```

**输出：**
```
CRITICAL NODES <node_count> <id1> <id2> ... EDGES <edge_count> <u1>-<v1> <u2>-<v2> ...
```

关键节点按 place_id 字典序，关键边按 `(min(u,v), max(u,v))` 字典序排序。

**示例：**
```
输入：CRITICAL
输出：CRITICAL NODES 2 P0005 P0006 EDGES 1 P0005-P0006
```

---

### 19. QUIT

退出程序。

```
QUIT
```

**无输出。**

## 拓展任务命令（仅在你做对应拓展时实现）

下面的命令**只在你做了《扩展与加分要求》中的对应拓展任务时**才需要实现，必做部分不会用到。如果你没做这些拓展，就不需要实现这些命令。

### X1. SHORTEST\_K（拓展任务 1：分层图最短路径）

```
SHORTEST_K <from_id> <to_id> <K>
```

**参数：**
- `from_id`、`to_id`：起终点 place\_id
- `K`：最多可用的加速通行券张数，整数，`0 ≤ K ≤ 10`

**输出：**

```
PATH <total_time> K_USED <k_used> NODES <id1> <id2> ... FAST <count> [<u1>-<v1> <u2>-<v2> ...]
```

- `<total_time>`：所选路径上所有边的实际耗时之和（用券的边按加速后耗时计入）；
- `<k_used>`：实际用了几张券（`0 ≤ k_used ≤ K`）；
- `NODES`：完整路径的 place\_id 序列；
- `FAST <count> ...`：用券的边列表，按 `(min(u,v), max(u,v))` 字典序排序。`count = 0` 时 FAST 后面无内容。

若不可达：`NO_PATH`

若起终点不存在：`ERROR place_not_found`
