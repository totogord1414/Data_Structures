# 扩展与加分要求（共 15 分）

拓展任务**不强制**，可以一个都不做（不影响必做 75 分 + 报告 10 分），也可以三个都做（拿满 15 分）。**做哪个拿哪个分**，每项独立计分。

> 与必做一样，每个拓展任务都对应**一组强制的评测接口**（命令名 + 输入参数 + 输出格式），见各项下方"评测接口"小节，以及《命令接口规范》"拓展任务命令"部分。其他实现细节（算法选择、内部结构、是否再做交互界面等）**完全自由**。

## 拓展 1：共享单车（9 分）

### 1.1 场景

校园内提供 **共享单车 / 接驳车券**：每张券允许在某一条边上启用"加速通行"，使该边的耗时从 `walk_time` 缩短为 `ceil(walk_time / 3)`（骑行/接驳速度约为步行的 3 倍）。每位学生有最多 **K 张券**（`0 ≤ K ≤ 10`），每张券绑定一条边，**不必用满**。

### 1.2 任务

给定起点 `from`、终点 `to`、券数 `K`，求**最短到达时间**及对应路径，并报告实际用了几张券、用在了哪些边上。仅考虑 `status = open` 的边。

### 1.3 评测接口（强制）

```
SHORTEST_K <from_id> <to_id> <K>
```

**输出：**
```
PATH <total_time> K_USED <k_used> NODES <id1> <id2> ... FAST <count> [<u1>-<v1> <u2>-<v2> ...]
```

- `<total_time>`：所选路径上所有边的实际耗时之和（用券的边按加速后耗时计入）；
- `<k_used>`：实际用了几张券（`0 ≤ k_used ≤ K`）；
- `NODES`：完整路径的 place\_id 序列；
- `FAST <count> ...`：用券的边列表，按 `(min(u,v), max(u,v))` 字典序排序。`count = 0` 时 FAST 后面无内容。

不可达：`NO_PATH`

起终点不存在：`ERROR place_not_found`

### 1.4 评分

| 维度 | 分值 |
|------|------|
| 算法正确性（K=0 / 中等 K / 边界 K=10 都正确） | 5 |
| 路径回溯 + 输出用券位置 | 2 |
| 报告中说明你的实现思路与复杂度 | 2 |

## 拓展 2：自定义微型数据集 + 对抗样例（3 分）

### 2.1 场景与任务

提交两组你**自己构造**的图数据，用于检验项目实现。所有 CSV 必须严格符合本项目的 schema（见《项目说明》§6）。

### 2.2 微型数据集

- 12~20 个地点、20~40 条道路；
- `places.csv` / `roads.csv` 完整、能正确加载；
- 附**说明文档**：
  - 数据来自哪里？（真实场景或自行构造，可以是任意场景：校园、商场、机场、医院、景区、地铁站等）
  - 为什么这样建模？关键约束/特点是什么？
  - 这个数据集想检验什么功能或边界情况？

### 2.3 对抗样例

- 至少**一组**能让错误实现或低效实现暴露问题的图数据（CSV 文件）；
- 附说明：
  - 为什么这组数据特殊？
  - 它能卡住哪类错误实现？
  - 你的程序如何正确处理？

### 2.4 评分

| 维度 | 分值 |
|------|------|
| 微型数据集 + 说明 | 1.5 |
| 对抗样例 + 说明 | 1.5 |

雷同提交（多个学生交同一份）不计分。

## 拓展 3：图形化界面（3 分）

### 3.1 场景与任务

把项目从命令行升级为图形化界面。任意图形库 / Web 技术皆可（Qt、Dear ImGui、HTML+canvas、D3、PyQt + matplotlib、Graphviz 导出 SVG 等）。

### 3.2 必须展示的内容

- 节点位置（手动布局 或 自动布局均可）；
- 边连接，区分 `open` / `closed`；
- 最短路径在图上高亮；
- 关键节点 / 关键边在图上标识。

### 3.3 提交

- 截图（至少 3~5 张，覆盖上述必须展示项）；
- 演示视频（30 秒~3 分钟，可选但推荐）；
- 简要技术说明（用什么库、怎么编译/运行）。

> 注意：图形化界面是**额外**做的，**不能替代必做的 CLI 评测接口**——必做的命令行入口仍需保留。

### 3.4 评分

| 维度 | 分值 |
|------|------|
| 必须展示项全部达标 | 2 |
| 在必须展示之外有亮点（自主发挥） | 1 |
